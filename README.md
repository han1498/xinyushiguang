## 🌟 产品理念

### 核心定位
「心语拾光」不是又一个聊天机器人，而是一款**个人信息资产化工具**。用户只需要像和朋友聊天一样自然表达，AI会在后台自动识别、分类、结构化，让碎片化信息沉淀为可管理的数字资产。

### 三大设计原则

| 原则 | 描述 | 核心价值 |
|------|------|----------|
| **零摩擦输入** | 聊天即输入，无需手动切换APP录入 | 把"录入成本"降到趋近于零 |
| **可感知反馈** | 颜文字状态栏实时反映AI理解状态 | 消除AI黑盒感，建立信任闭环 |
| **自进化沉淀** | 越用越懂你，用户画像自动丰富 | 产品价值随使用时长递增 |

## ✨ 功能特性

### 💬 AI 陪伴聊天
- 温柔治愈的"百合花精灵"人设，自然温暖的对话风格
- 支持自定义AI身份、能力边界、对话规则
- 实时情绪识别 + 颜文字状态栏反馈
- Markdown 格式富文本回复展示

### ✅ 待办事项管理
- 日历视图 + 每日待办清单
- 支持永久有效 / 自定义有效期两种模式
- 每日最多4个待办，保持专注（可配置）
- 全部完成时的鼓励机制，正向激励

### 📝 智能备忘中心
- **记忆库**：AI自动提取的用户偏好、性格、重要事件
- **日历事件**：自动识别对话中的日程安排
- **提醒事项**：自动识别需要提醒的任务
- 用户档案：姓名、生日、性格标签，个性化AI回复

### 🌷 其他功能
- 广场模块（社区内容展示）
- 多主题素材配置
- 本地数据持久化存储
- PWA Web 版本支持

## 🏗️ 系统架构

### 数据流转图

```
用户输入 → 情绪识别 → 状态栏反馈
    ↓
  AI对话引擎 ──→ 聊天记录持久化
    ↓（并行）

    ├─→ 提醒事项库
```

## 🚀 快速开始

### 环境要求
- 微信开发者工具
- DeepSeek API Key（或兼容 OpenAI 格式的 API 服务）

### 安装与运行

1. **克隆项目**
```bash
git clone <repository-url>
cd 心语拾光
```

2. **导入微信开发者工具**
   - 打开微信开发者工具
   - 选择"导入项目"
   - 选择项目根目录
   - 填入 AppID（或使用测试号）

3. **配置 API 服务**
   - 进入小程序 → 点击"设置"
   - 填写 API Base URL 和 API Token
   - 默认使用 DeepSeek API：`https://api.deepseek.com/v1/chat/completions`

4. **开始使用**
   - 在聊天页输入对话内容
   - AI自动提取的信息可在"备忘"页查看

## 📊 核心数据模型

### 待办事项 (TodoItem)
```javascript
{
  id: "t_xxx",        // 唯一标识
  date: "2024-01-01", // 所属日期
  content: "内容",     // 待办内容
  done: false,        // 完成状态
  duration: "permanent|custom", // 有效期类型
  expireDate: "",     // 过期日期（custom 时有效）
  createdAt: 1234567890 // 创建时间戳
}
```

### 记忆条目 (MemoryItem)
```javascript
{
  id: "m_xxx",
  content: "记忆内容",
  category: "preference|habit|event|personal", // 分类
  createdAt: 1234567890
}
```

### 用户画像 (UserProfile)
```javascript
{
  name: "",        // 昵称
  birthday: "",    // 生日
  personality: ""  // 性格描述
}
```

## 🧠 AI 能力说明

### 1. 对话生成
- 基于大语言模型的自然对话
- 支持自定义人设、边界、规则
- 系统提示词动态构建（身份 + 边界 + 用户画像 + 记忆）

### 2. 情绪识别
- 关键词匹配 + 规则引擎
- 6 种情绪分类：开心、温暖、难过、惊讶、鼓励、思考
- 实时颜文字反馈（20+ 种颜文字表情）

### 3. 信息提取
- **并行提取**：不影响主对话流程
- **三类信息**：日历事件、提醒事项、用户记忆
- **容错设计**：提取失败静默降级，不中断主流程
- **去重机制**：基于内容指纹的自动去重

---

## 🔒 容错与异常处理

采用三级错误策略：

| 级别 | 场景 | 处理方式 |
|------|------|----------|
| 🔴 可感知错误 | 网络异常、服务不可用 | 友好提示 + 错误码 + 用户可理解文案 |
| 🟠 引导式纠错 | 未配置 API Key、人设缺失 | 引导用户去设置页完成配置 |
| 🟢 静默降级 | 信息提取失败、格式解析异常 | 不影响主流程，仅降级增值功能 |
